## Context

## Decision

因为我们并不是要做一个可以运营的产品，所以定义为 POC 项目，实现的优先级如下

1. 满足钱包 POC 业务需求，包括如下：
    - DID 协议
    - 可验证声明，健康码
    - 转账交易
2. 密码学组件
    - 验证签名需要用到
3. 同步服务
    - 获取交易列表需要用到
    - 获取交易详情需要用到

可以暂不实现的是
1. 消息队列
    - 可以实现监听区块链消息，完成复杂的计算，推送等服务
2. 托管私钥服务
    - 可以完成托管服务，但这个一般针对商家，在钱包服务中不是必须的
3. 数据库读写分离和进一步优化
    - 链上交易多的化可能会有很多插入操作
4. 邮件和短信验证
    - 登录注册的时候进行验证
5. 去中心化登录
    - 可以作为另一种第三方登录的形式
